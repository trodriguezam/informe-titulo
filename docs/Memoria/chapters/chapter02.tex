\chapter{Solución Planteada}
\label{ch:solucion-planteada}

\section{DESCRIPCIÓN DE LA SOLUCIÓN}

Frente a la falta de fantomas anatómicamente diversos para el entrenamiento de procedimientos de emergencia como la cricotiroidotomía, se propone un sistema que automatiza la generación de fantomas de cuello personalizados a partir de CTs de pacientes reales. La solución reemplaza el flujo manual de segmentación, corrección y exportación, el cual es lento y dependiente de un operador con experiencia en herramientas de modelamiento 3D, por un pipeline reproducible que un estudiante o docente de medicina puede ejecutar sin conocimientos de procesamiento de imágenes ni de impresión 3D.\\

El sistema se organiza en cuatro etapas: a) ingreso de la TC en formato NIfTI; b) segmentación automática de las estructuras anatómicas relevantes para la vía aérea y el cuello (vértebras cervicales, cartílagos, tráquea, músculos, glándulas, tejidos blandos, entre otras); c) exportación de cada estructura segmentada a un archivo STL independiente, con asignación de color según el tipo de tejido; y d) preparación de los STL en PrusaSlicer para su impresión multimaterial en una impresora Prusa XL. Estas cuatro etapas se exponen al usuario final a través de un módulo personalizado de 3D Slicer, de modo que la complejidad del pipeline (ejecución remota en GPU, consolidación de segmentos, mapeo de colores) queda oculta detrás de una interfaz simple.\\

Para la etapa de segmentación automática se evaluaron comparativamente tres alternativas (TotalSegmentator, MedSAM y YOLO) sobre una base de diez casos de cuello con segmentación manual de referencia realizada por un radiólogo. Como resultado de esta evaluación se seleccionó \citep{wasserthal2023totalsegmentator} como motor de segmentación de la solución, dado su desempeño consistentemente superior frente a las estructuras de interés del proyecto.\\

A diferencia del flujo de comparación de modelos, que exige mantener una correspondencia estricta con las estructuras anotadas manualmente por el radiólogo, el pipeline del fantoma adopta un criterio más permisivo: se conserva cualquier estructura detectada con contenido no vacío, independientemente de si fue parte del conjunto de referencia. Este enfoque hace que el sistema sea agnóstico a la imagen de entrada, permitiendo procesar casos nuevos sin necesidad de una segmentación manual previa que sirva de comparación.\\

El cómputo de la segmentación, el cual presenta un uso intenso de GPU, se ejecuta de forma remota en una máquina dedicada, comunicándose mediante SSH sobre una red privada. Esta decisión de arquitectura permite que el módulo de Slicer corra en un equipo sin GPU dedicada, ampliando el alcance de la herramienta a los computadores típicamente disponibles en un contexto docente.\\

\section{REQUISITOS}

\subsection{Requisitos funcionales}

\begin{enumerate}
  \item Permitir el ingreso de una tomografía computarizada de cuello en formato NIfTI desde el módulo de 3D Slicer.
  \item Ejecutar la segmentación automática de las estructuras anatómicas relevantes para la vía aérea y el cuello, delegando el cómputo a una máquina remota con GPU mediante conexión SSH.
  \item Consolidar los segmentos obtenidos descartando únicamente aquellos sin contenido (máscaras vacías), de manera agnóstica a la imagen de entrada.
  \item Exportar cada estructura segmentada a un archivo STL independiente, asignando un color según el tipo de tejido al que pertenece.
  \item Permitir el corte y reorientación de las estructuras exportadas mediante planos interactivos, previo a su impresión.
  \item Generar un reporte que indique qué estructuras fueron detectadas y cuáles no, para cada caso procesado.
  \item Permitir la visualización de las superficies 3D generadas dentro de 3D Slicer antes de exportar a impresión.
\end{enumerate}

\subsection{Requisitos operacionales}

\begin{enumerate}
  \item El sistema debe operar como un módulo instalable dentro de 3D Slicer (versión 5.10 o superior), sin requerir conocimientos de programación por parte del usuario final.
  \item El equipo del usuario no requiere GPU dedicada: el cómputo intensivo de la segmentación se ejecuta de forma remota en la máquina del profesor guía, mediante una red privada (Tailscale) y autenticación por llave SSH.
  \item La máquina remota debe contar con TotalSegmentator instalado y, para las tareas que lo requieran (\textit{tissue\_types}), con la licencia académica correspondiente activada.
  \item Los archivos STL resultantes deben ser compatibles con PrusaSlicer para su preparación e impresión en una impresora Prusa XL multimaterial.
  \item El tiempo de procesamiento esperado por caso, incluyendo transferencia de archivos y segmentación remota, no debe superar los 15 minutos.
\end{enumerate}

\subsection{Atributos de calidad}

\begin{enumerate}
  \item \textbf{Precisión:} las estructuras críticas para la observación de la cricotiroidotomía (vía aérea, cartílagos, vértebras cervicales) deben alcanzar un coeficiente de Dice promedio igual o superior a 0.90 frente a la segmentación de referencia del radiólogo.
  \item \textbf{Usabilidad:} el módulo debe ocultar la complejidad técnica del pipeline (ejecución remota, consolidación de segmentos, mapeo de colores) detrás de una interfaz simple, utilizable por personal médico sin formación en procesamiento de imágenes.
  \item \textbf{Portabilidad:} el módulo debe funcionar tanto en Windows como en macOS, sin depender de hardware gráfico especializado en el equipo local.
  \item \textbf{Mantenibilidad:} la separación entre el módulo de segmentación (FantomaModule) y el módulo de corte y exportación (CorteModule) debe permitir modificar uno sin afectar el funcionamiento del otro.
\end{enumerate}

\subsection{Restricciones}

\begin{enumerate}
  \item Los diez casos de evaluación y la segmentación manual de referencia (ground truth) del radiólogo se mantienen únicamente en el equipo local del equipo de trabajo; la infraestructura remota se utiliza exclusivamente para la segmentación del pipeline del fantoma.
  \item El uso de la tarea \textit{tissue\_types} de TotalSegmentator está condicionado a la disponibilidad de una licencia académica activa.
  \item La impresión multimaterial está restringida al equipo disponible en el proyecto (Prusa XL), por lo que el mapeo de colores y materiales se diseña en función de sus capacidades.
\end{enumerate}
